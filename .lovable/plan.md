# Inspections — Trash & Quick-View Actions

Mirror the pattern already used by Equipment / Instruments / PQRs / Projects (which have `*.trash.tsx` routes and soft-delete) and apply it to Inspections.

## 1. Soft-delete ("Move to Trash")

- Add `deleted_at timestamptz` (and `deleted_by uuid`) to `public.inspections` via migration.
- Update RLS / list query to filter `deleted_at IS NULL` by default.
- Row action: trash icon (`Trash2`) in a new right-aligned **Actions** column. Click → confirm dialog → sets `deleted_at = now()`, writes an `inspection_events` audit row (`kind: 'trashed'`), toasts, refetches.
- New route `src/routes/app.inspections.trash.tsx`:
  - Lists inspections where `deleted_at IS NOT NULL`.
  - Per-row **Restore** (clears `deleted_at`) and **Delete permanently** (hard delete, confirm dialog).
- Header link "Trash" on the main Inspections page (matches Equipment/Instruments pattern).

## 2. Eye icon → Inspection Details

- In the same Actions column, add an `Eye` icon button linking to the existing detail route `/app/inspections/$inspectionId` (the row link already navigates there, but an explicit affordance is requested).
- Keep the existing row click behaviour intact.

## Files

- `supabase/migrations/<ts>_inspections_soft_delete.sql` — add `deleted_at`, `deleted_by`; index on `deleted_at`.
- `src/routes/app.inspections.tsx` — Actions column with Eye + Trash buttons, filter `deleted_at IS NULL`, Trash link in header.
- `src/routes/app.inspections.trash.tsx` — new trash view with restore / permanent-delete.
- `src/routeTree.gen.ts` — regenerated by the router plugin.

## Out of scope

No changes to the inspection workflow, schema beyond soft-delete columns, or other modules.
